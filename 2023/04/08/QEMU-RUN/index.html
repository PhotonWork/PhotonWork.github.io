PCI设备简介

什么是PCI设备，关于PCI设备，小编也很好奇….

不扯皮了,PCI设备简而言之就是插在PCI插槽上面的一些设备,包括比如声卡 网卡之类的.它们都遵循PCI总线标准

既然有了设备,我们首先想到的肯定是如何与其交互的问题

我们在操作系统课(还是计组来着,我忘了)上学过,CPU与外设进行交互通常分为两种类型:

  • MMIO:也就是内存映射IO,外设和内存共享地址空间,我们可以像读写内存一样读写外设内容
  • PMIO:也就是端口映射IO,外设和内存的地址空间相独立,CPU需要用一些专门的指令与外设进行交互(比如INTEL的CPU使用IN和OUT指令)

我们先有这么一个概念,后面及题目中会有详细介绍如何交互的过程

不过大概许多人和我一样都有一些疑问:PCI设备是如何被CPU发现的,操作系统如何管理这些PCI设备,我们如何查看电脑上插了哪些设备….这些问题后面也许会提到一些吧(x

PCI设备配置空间

每个PCI设备都有一个配置空间,介绍了关于这个设备的一些信息,其大小为256字节,头部64字节是PCI标准规定的,

一般我们关注Device ID和Vendor ID就行了

还有一个比较重要的是Base Address Registers(BAR)这块,它记录了设备用到的地址空间类型\基址以及其他属性

BAR的最后一位标识了地址空间类型,也就是前面简要提到的MMIO以及PMIO(为0时为MMIO)

而其他字段根据图也可以很清楚的看到,比较重要的就是高位的基址(Base Address)

QOM(Qemu Object Module)模型

QOM是QEMU提供的一套面向对象编程模型(虽然QEMU是用C来写的，但是利用QOM我们也能进行面向对象的一些操作)，用来实现一些设备

我们可以通过这个模型，来自己编写QEMU中的设备，它主要由四个组件构成

  • Type 定义一个类
  • Class 存储类静态数据和函数指针
  • Object 存储动态数据，动态分配一个类的实例
  • Property 动态对象数据的访问器

TypeInfo

TypeInfo是用户来定义一个Type的结构体，我们定义了一个TypeInfo之后，调用type_register(TypeInfo)或者type_register_static(TypeInfo)函数，就能生成一个TypeImpl实例

当我们想定义一个类的时候，我们需要定义一个TypeInfo类型的变量

Class

我们可以通过Class结构体来定义类的静态内容，比如静态成员，函数表之类的，它应当继承自某个Class类型

ObjectClass是所有类的基类

当所有的qemu总线，设备等的type_register_static执行完成之后(也就是他们的TypeImpl实例被创建成功之后)，qemu会在type_initialize函数中去实例化对应的ObjectClasses

比如我们可以定义一个Test的Class

在类的定义中父类都在第一个字段，让父类和子类可以直接转换。类初始化时会先初始化它的父类

这样定义之后，我们也需要在之前的类定义那里去修改一下与Class相关的构造函数等

Object

Object类型用来表示类的一个实例对象。刚刚所说的TypeInfo以及ObjectClass只是定义的类型，而非具体的设备，我们之前所介绍的TypeInfo结构体中的有两个函数指针instance_init以及class_init，前者负责初始化具体Object结构体,而后者负责初始化ObjectClass结构体

这个类型我们同样需要定义，其继承自某个Object

同ObjectClass类似，所有Object结构体类型的基类都是Object结构体

同样，我们在类定义那里需要修改一下与Object相关的东西

PCI的内存空间

qemu中使用MemoryRegion来标识内存空间(定义于include/exec/memory/h)。使用MemoryRegionOps结构体来定义对内存操作的一些内容。对于MMIO以及PMIO操作都需要相应的MemoryRegionOps结构体，在此结构体中我们可以定义对应的read/write回调函数
