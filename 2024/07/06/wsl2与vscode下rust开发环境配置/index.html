wsl2与vscode下rust开发环境配置

Created2024-07-06|Updated2024-07-06
|Word count:251|Reading time:1min

之前写代码的模式一直都是vscode编辑，然后wsl2开个terminal编译。因为大多写得都是脚本之类的所以也无伤大雅。但是如果想写一些中大型项目还是有vscode的一些插件支持会好一点，这里记录一下配置过程。

首先连接到wsl,查看->命令面板，直接连就可以了

然后在扩展里面安装一下插件，主要是rust-analyzer

这里出了一点问题，glibc版本不太对，ubuntu18.04用的glibc没那么新。

有几个办法:

  1. 降低rust-analyzer版本

  2. 换个高版本的ubuntu连接

  3. 手动安装rust-analyzer，然后指定路径(“Bootstrap Error” rust-analyzer requires glibc >= 2.28 in latest build. · Issue #11558 · rust-lang/rust-analyzer (github.com) )
  4. 因为我本地有高版本glibc，直接patchelf应该可行

这里直接选择换ubuntu20.04连，

换成20.04即可

然后就能享受一些便捷操作了。User Manual (rust-analyzer.github.io) 文档有详细操作。
